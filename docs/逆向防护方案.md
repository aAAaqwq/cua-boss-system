# 本地自动化「防逆向」分析与方案（存档）

> 结论先行：**现在不必动**（T0 君子锁 + 云端锁够用）；**等商业化收费时直接上 T1「AI 网关」**——
> 它不只是防逆向，而是「把账号变成产品命脉 + 保护提示词秘方 + 能计费」的商业基础设施，顺带把逆向问题一并解决。
> 配套：[权限架构.md](权限架构.md) §7 绕过边界。

## 1. 先想清楚：在保护什么？
真正值钱的**不是**编排代码（点击→抓取→回复，commodity，谁都能重写），而是：

| 资产 | 现状 | 风险方 |
|------|------|--------|
| 候选人**数据** | ✅ 已锁死（云端 RLS，服务端强制，绕不过）| — |
| **提示词**（system/scoring/intent prompt，= 回复质量&评分秘方）| ❌ 明文随包发出 | 抄袭者 |
| **AI 能力**（DeepSeek 调用）| ❌ 本地用本地 key 直连 | 白嫖额度者 |

→ 问题不是「怎么混淆 Python」，而是「**怎么让没账号的人用不了智能能力、也偷不到提示词**」。
→ **混淆编排逻辑 = 低 ROI**（可破 + 保护的是 commodity 代码 + 增加打包/调试痛苦）。

## 2. 是否必要：看阶段
- **现在（验证期 / 少量用户 / 熟人）**：当前「君子锁」(`require_account`) + 云端锁**够了**，过度工程拖累迭代。
- **开始收费 / 有竞品 / 盗版有真实成本时**：值得投入 T1。

## 3. 最佳方案：AI 网关（服务端代理）—— 一箭四雕
把 DeepSeek 调用 + 提示词搬到服务端（复用现有 Cloudflare Worker + Supabase 鉴权）：
```
本地 chat_reply/scoring ──带 user token──► AI 网关 Worker
                                            ① 校验账号有效(Supabase)
                                            ② 服务端持 DeepSeek key + 提示词
                                            ③ 调 DeepSeek → 返回结果
                                            ④ 计量 / 限流 / 可吊销
```
**四个收益**：
1. **账号变强制**：没有效账号 → 拿不到智能回复/评分 → 产品被掏空，改本地代码也救不回（服务端强制，真绕不过）。
2. **提示词永不出服务器**：秘方用户拿不到。
3. **成本可控 + 可计费**：DeepSeek 费用集中，按用户计量、可按调用收费。
4. **可远程吊销**：停账号 → AI 立即失效。

**代价**：多一跳延迟（国内→Cloudflare→DeepSeek）、你承担 DeepSeek 成本（需计费模型）、多一个服务组件 + 单点依赖。

## 4. 分层路线图（按阶段加，别一步到位）
| 层 | 做什么 | 何时 | ROI |
|----|--------|------|-----|
| **T0（现状）** | 君子锁 `require_account` + 云端 RLS 锁 | 已完成 | 够用 |
| **T1（收费时）** | **AI 网关**：提示词 + DeepSeek 调用服务端化，本地改走代理 + 计量 | 商业化 | ⭐最高 |
| **T2（盗版严重）** | + 代码混淆(Nuitka) + 每步关键操作 token 校验(可吊销) | 真有人盗 | 递减 |

## 5. T1 实施要点（将来动手时的清单）
- 新 Worker `bole-ai-proxy`：secret 持 `DEEPSEEK_API_KEY` + 提示词（从 `config/*.md` 迁入服务端）；endpoints `/ai/reply` `/ai/score` `/ai/intent`。
- 鉴权：复用 `bole-admin-api` 同款（校验 Supabase user token → 账号有效 → 放行）。
- 本地：`app/chat_reply.py` / `app/scoring.py` 增「网关模式」——有 `AI_GATEWAY_URL` 则走代理(带 user token)，否则回退本地 key（拥有者开发用）。
- 计量：Worker 记录每用户调用数 → 限流 / 配额 / 计费基础。
- 注意：延迟与失败需 best-effort 降级（网关挂 → 提示而非崩）；提示词版本管理。

## 6. 当前决定（2026-06-27）
**保持 T0，不额外加固**。理由：验证期盗版非主要矛盾；云端数据已固若金汤；本地编排是 commodity。
**触发 T1 的信号**：开始对外收费 / 出现抄袭或白嫖 / 需要按用量计费。届时按 §5 实施。
