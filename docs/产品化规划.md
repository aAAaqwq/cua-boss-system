# 产品化规划（feat/productize）

> 把 cua-boss-system 从「一套脚本」做成「给不懂技术的 HR 的产品」——伯乐 AI 招聘助手。
> 配套：[使用手册.md](使用手册.md)、[数据安全说明.md](数据安全说明.md)、[../SKILL.md](../SKILL.md)、issue #5（Tauri 桌面 App）。

## 一、定位与形态

- **是什么**：一个 macOS 桌面 App「**伯乐**」——HR 双击即用，不碰终端、不懂代码。
- **核心价值**：AI 评分 + 智能沟通 + 全流程编排 + 人格化助手，帮 HR 从「一次打招呼」走到「全流程自动化」。
- **形态三层**（全藏在 App 里）：
  ```
  Tauri 壳(品牌/看板/问伯乐) → 本地引擎(Python 脚本 + cua-driver + SQLite) → 云端(Supabase 多租户 RLS + 许可门禁)
  ```

## 二、合规边界（先于功能，必须守）

> ⚠️ 公开售卖「未授权 GUI 自动化 + 批量采集第三方平台 PII」有 ToS / PIPL / PII 出境三重风险。
> 产品化走**合规路径**：

1. **人在环路**：定位为「招聘官辅助工具、人工确认后操作」，弱化「全自动爬取」叙事。
2. **PII 本地化**：候选人敏感数据**不出境**（当前在美区 Supabase → 需迁中国区或本地优先）。
3. **告知-同意**：除「使用即同意」（对*用户*，已加）外，补**对候选人**的告知-同意机制。
4. **责任切割**：使用条款（已加雏形）+ 用户须自证合法采集授权。
5. **专业法律意见**：ToS + PIPL 各做一次专业咨询（非工程能替代）。

| 阶段 | 目标 | 关键交付 | 状态 |
|------|------|----------|------|
| **P0 内核** | 命令行可跑全流程 + 云端多租户 | 脚本/评分/话术/Supabase RLS/许可门禁 | ✅ 已有 |

## 四、本次落地（P1）

### 伯乐 agent（DeepSeek 驱动，能真实操作系统）
- **`app/bole_agent.py`**：`chat()`（纯对话）+ **`run_agent()`（工具调用循环）**；system prompt = 人设三件套（IDENTITY/SOUL/AGENTS）+ 应对手册（faq）+ 能力清单；用 `.env` 的 `DEEPSEEK_API_KEY`，零依赖纯 urllib，带重试。
- **`desktop/bole_tools.py`**（P2 增强）：伯乐不只是聊天，是**能真干活的 agent**——工具集读真实候选人库（get_dashboard/top_candidates/find_candidate）、跑真实脚本（run_task 起后台任务/job_status）、约面试（schedule_interview）。**动作类默认 dry_run 预览+请用户确认**，绝不擅自真操作。
- **`scripts/bole.py`**：交互 REPL + `--ask` 单轮 + `--json`（给 App/前端调用）。
- 这是桌面 App「问伯乐」对话框的内核：不懂技术的 HR 直接打字让伯乐**查真实数据/真干活**（聊天里 🔧 透明化展示实际调用的工具）。

### 装机自检（onboarding 内核）
- **`scripts/doctor.py`**：自动测 Python/cua-driver/Chrome/.env/配置/登录/DB（`--json` 供前端），并列出「必须人工确认」项（TCC 权限、BOSS 登录、Apple 事件 JS、防睡眠）。
- 既是排障工具，也是将来 Tauri App「首次分步引导」的内核（点不过不让往下走）。

## 五、本次落地（P2 桌面化）

不装 Rust/Tauri 也能先"跑起来、可测试"：桌面端做成**本地 Web App**——一个只绑
`127.0.0.1` 的纯标准库 HTTP 服务，把浏览器 UI 桥接到既有脚本与 `candidates.db`。
**这一层就是将来 Tauri 壳的 WebView 前端**（Tauri 打包时直接复用 `desktop/ui` 与
这些 `/api` 接口，不重写），因此是「今天可测 + 明天可打包」的增量路径。

### 交付物
- **`desktop/server.py`**：本地服务（零依赖）。接口：
  `GET /api/doctor`（自检）、`GET /api/dashboard`（看板统计+评分榜，只读库）、
  `GET·POST /api/config`（读/写 DeepSeek key/模型/接口/云同步开关到 `.env`）、
  `POST /api/bole`（伯乐 agent：工具调用后作答，返回 actions）+ `POST /api/bole/suggest`
  （按回复动态生成快捷回复）、`POST /api/run` + `GET /api/job/{id}`
  （白名单任务 greet/collect/chat/pipeline 后台跑 + 日志轮询）、
  `GET /api/candidate` + `/api/resume`（候选人详情+简历PDF）、`POST /api/shutdown`（优雅停服）。
  **登录门禁**：涉 PII/配置/操作接口未登录一律 401；**CSRF** 校验 Origin + 只解析 application/json。
- **`desktop/ui/`**：单页控制台（登录门禁 → 看板/候选人详情弹层 / 操作台实时日志 / Telegram 风问伯乐 / 设置），伯乐品牌视觉。
- **打包**：`desktop/build_app.sh`（osacompile 生成原生 `伯乐.app`）/ `desktop/伯乐.command`（nohup 脱离 + Chrome 应用窗口模式）。

### 亮点：DeepSeek key 可在桌面端配置
`设置` 页直接填 key → 写 `.env` **并同时** `os.environ` 生效（`_get_deepseek_config`
每次读 env、env 优先级高于 .env）→ **无需重启即时生效**；掩码回显、掩码值再保存不会
误覆盖真 key；写入保留其余行与注释、原子替换。用户不再需要碰终端/手改 `.env`。

### 安全（本次一并修复 · 见 swarm 审查）
- **HIGH·JS 注入**：候选人姓名等外部可控数据拼进 `execute_javascript` 时原先只转义
  单引号、不转义反斜杠（`\'` 可逃逸字符串上下文）。新增统一 `_js_str()`（先反斜杠、
  再引号，另转义反引号/`${`/换行），替换 `cua_collect.py` 全部 5 处拼接点。
- **MEDIUM·SSRF**：简历下载 URL 的 iframe-direct/embed 分支原样返回站内任意 `src`，
  urllib 兜底默认含 `FileHandler` 可读 `file://`。新增 `_is_safe_pdf_url()` 白名单
  （仅 https + `zhipin.com`/`bosszhipin.com`），非法地址不进任何一级下载链路。

## 六、P3 规划（商业化 · 待办）

> 目标：从「能用的桌面端」到「可售卖、可维护、合规」的产品。按优先级排列。

### 6.1 分发与打包（先解决"能装")
- **Tauri 壳**：装 Rust 工具链，用 Tauri 包 `desktop/ui` 为原生 `.app`；Rust 侧
  仅做「拉起本地 Python 引擎 + 开窗口」，前端/接口零改动。
- **Python 运行时内嵌**：随包带 `python-build-standalone`，用户无需自装 Python。
- **签名 + 公证**：Apple 开发者证书（$99/年）签名 + notarize，产出 `伯乐.dmg`，
  过 Gatekeeper（否则用户要右键→打开）。
- **cua-driver 依赖**：随包分发或首启引导安装 + TCC 权限分步引导（复用 doctor.py）。

### 6.2 计费 / 订阅
- **分版**：自用 / 团队 / 按量三档。
- **订阅校验接许可门禁**：现有登录门禁（Supabase 许可）上加订阅态字段，过期即降级只读。
- **DeepSeek 网关化**：key 收到服务端做共享网关（用户不填 key、按量计费、可轮换、
  可监控失效），替代「每台机各配 key」；桌面端设置页保留「用自己的 key」高级选项。

### 6.3 租户自助开通
- 管理台（`admin-api/` + `web/admin.html`）加自助注册 → 建 Supabase 用户 + 初始配额，
  免人工发账号；下发引导邮件（登录 + 装机自检链接）。

### 6.4 PII 合规（法律高压项，售卖前必须）
- **数据本地化**：候选人 PII 从美区 Supabase 迁**中国区**或改**本地优先**（默认不出境，
  云同步改为可选、明确告知）。
- **上云字段脱敏**：`resume_path` 现在同步的是本机**绝对路径**（泄露 OS 用户名/目录），
  改为只同步文件名/相对路径（swarm 审查 LOW）。
- **对候选人告知-同意**：补候选人侧告知机制（现仅对*用户*有「使用即同意」）。
- **专业法律意见**：ToS + PIPL 各做一次专业咨询。

### 6.5 打磨（体验/健壮）
- 密码不走 CLI 明文参数（`cloud_sync.py` 引导用 `getpass`，审查 MEDIUM）。
- 二次确认 Supabase 端 RLS 按 `tenant_id = auth.uid()` 强隔离（审查 INFO）。
- 桌面端：任务运行态持久化（刷新不丢日志）、多任务队列、结果通知。

## 七、风险登记

| 风险 | 影响 | 缓解 |
|------|------|------|
| 平台 ToS / 封号 | 产品命脉系于 BOSS | 弱自动化 + 人在环路 + 多平台备选 |
| PIPL / PII 出境 | 法律高压 | 数据本地化 + 告知同意 + 法律意见 |
| 反检测军备竞赛 | 维护成本高 | 稳定性已在投入（AX/下载健壮性、CGEvent 兜底） |
| DeepSeek key 失效 | 全 AI 层降级 | 网关化 + key 监控；**当前 .env 的 key 已 401 失效，需更换** |
