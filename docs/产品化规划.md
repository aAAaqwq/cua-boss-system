# 产品化规划（feat/productize）

> 把 cua-boss-system 从「一套脚本」做成「给不懂技术的 HR 的产品」——伯乐 AI 招聘助手。
> 配套：[使用手册.md](使用手册.md)、[数据安全说明.md](数据安全说明.md)、[../SKILL.md](../SKILL.md)、issue #5（Tauri 桌面 App）。

## 一、定位与形态

- **是什么**：一个 macOS 桌面 App「**伯乐**」——HR 双击即用，不碰终端、不懂代码。
- **核心价值**：AI 评分 + 智能沟通 + 全流程编排 + 人格化助手，帮 HR 从「一次打招呼」走到「全流程自动化」。
- **形态三层**（全藏在 App 里）：
  ```
  Tauri 壳(品牌/看板/问伯乐) → 本地引擎(Python 脚本 + cua-driver + SQLite) → 云端(Supabase 多租户 RLS + 许可门禁)
  ```

## 二、合规边界（先于功能，必须守）

> ⚠️ 公开售卖「未授权 GUI 自动化 + 批量采集第三方平台 PII」有 ToS / PIPL / PII 出境三重风险。
> 产品化走**合规路径**：

1. **人在环路**：定位为「招聘官辅助工具、人工确认后操作」，弱化「全自动爬取」叙事。
2. **PII 本地化**：候选人敏感数据**不出境**（当前在美区 Supabase → 需迁中国区或本地优先）。
3. **告知-同意**：除「使用即同意」（对*用户*，已加）外，补**对候选人**的告知-同意机制。
4. **责任切割**：使用条款（已加雏形）+ 用户须自证合法采集授权。
5. **专业法律意见**：ToS + PIPL 各做一次专业咨询（非工程能替代）。

## 三、阶段路线

| 阶段 | 目标 | 关键交付 | 状态 |
|------|------|----------|------|
| **P0 内核** | 命令行可跑全流程 + 云端多租户 | 脚本/评分/话术/Supabase RLS/许可门禁 | ✅ 已有 |
| **P1 助手化** | 伯乐对话驱动 + 装机自检 | `app/bole_agent.py`（DeepSeek）、`scripts/bole.py`、`scripts/doctor.py` | ✅ 本次 |
| **P2 桌面化** | Tauri App：看板 + 操作按钮 + 问伯乐 + 首次引导 | 见 issue #5；App 调本地脚本 + bole/doctor 的 `--json` | ⏳ 待办 |
| **P3 商业化** | 计费/订阅、租户自助开通、签名公证分发、PII 合规 | 订阅校验、`.dmg` 公证、数据本地化 | ⏳ 待办 |

## 四、本次落地（P1）

### 伯乐对话 agent（DeepSeek 驱动）
- **`app/bole_agent.py`**：`chat()/ask()`，system prompt = 人设三件套（IDENTITY/SOUL/AGENTS）+ 应对手册（faq）+ 能力清单；用 `.env` 的 `DEEPSEEK_API_KEY`，零依赖纯 urllib，带重试。
- **`scripts/bole.py`**：交互 REPL + `--ask` 单轮 + `--json`（给 App/前端调用）。
- 这是桌面 App「问伯乐」对话框的内核：不懂技术的 HR 直接打字让伯乐干活/答疑。

### 装机自检（onboarding 内核）
- **`scripts/doctor.py`**：自动测 Python/cua-driver/Chrome/.env/配置/登录/DB（`--json` 供前端），并列出「必须人工确认」项（TCC 权限、BOSS 登录、Apple 事件 JS、防睡眠）。
- 既是排障工具，也是将来 Tauri App「首次分步引导」的内核（点不过不让往下走）。

## 五、待办（P2/P3，不在本次）

- **Tauri 桌面 App**（issue #5）：壳 + 看板 + 操作面板 + 嵌入「问伯乐」+ 首次引导（调 `doctor.py --json` / `bole.py --json` / 各脚本）。
- **DeepSeek 网关化**：把 key 收到服务端做共享网关（用户不填 key、按量计费、可轮换），而非每台机配 key。
- **计费/订阅**：分版（自用 / 团队 / 按量），订阅校验接进许可门禁。
- **分发**：Apple 开发者证书（$99/年）签名 + 公证，产出 `伯乐.dmg`。
- **PII 合规**：数据本地化（迁中国区/本地优先）、对候选人告知-同意。

## 六、风险登记

| 风险 | 影响 | 缓解 |
|------|------|------|
| 平台 ToS / 封号 | 产品命脉系于 BOSS | 弱自动化 + 人在环路 + 多平台备选 |
| PIPL / PII 出境 | 法律高压 | 数据本地化 + 告知同意 + 法律意见 |
| 反检测军备竞赛 | 维护成本高 | 稳定性已在投入（AX/下载健壮性、CGEvent 兜底） |
| DeepSeek key 失效 | 全 AI 层降级 | 网关化 + key 监控；**当前 .env 的 key 已 401 失效，需更换** |
