# 伯乐 · 安装教程（从零到能跑）

面向不懂技术的 HR，一步步照做即可。全程约 15 分钟。遇到卡住看最后「常见问题」。

> 只支持 **macOS 12+**（不支持 Windows/Linux）。使用前请确保你有合法招聘授权，详见《[数据安全说明](数据安全说明.md)》。

---

## 准备清单

- 一台 Mac（macOS 12 及以上）
- 我们发给你的**登录账号**（邮箱 + 密码）
- 我们发给你的 **DeepSeek API Key**（决定 AI 回复/评分质量）
- BOSS 直聘的招聘者账号

---

## 第 1 步 · 装 Python（引擎运行环境）

打开「终端」App（聚焦搜索 `Terminal` 回车），粘贴运行：

```bash
python3 --version
```

- 显示 `Python 3.10` 或更高 → 已就绪，跳到第 2 步。
- 提示找不到 → 去 [python.org/downloads](https://www.python.org/downloads/) 下载最新版 macOS 安装包，双击安装。

## 第 2 步 · 装 cua-driver（驱动 Chrome 的核心）

cua-driver 是让伯乐能操作 Chrome 的底层工具，按我们提供的安装说明把它装到系统里
（装好后终端运行 `cua-driver --version` 能看到版本号即成功）。

> 若暂时没有，先跳过——启动伯乐后「装机自检」会明确告诉你缺什么。

## 第 3 步 · 装 Chrome 并登录 BOSS

1. 没装 Chrome 就去 [google.com/chrome](https://www.google.com/chrome/) 下载安装（有 Homebrew 也可 `brew install --cask google-chrome`）。
2. 打开 Chrome，登录你的 **BOSS 直聘招聘者账号**，保持登录状态。

## 第 4 步 · 开三项系统权限（关键，且只能手动开）

> ⚠️ macOS 出于安全设计，**不允许任何程序自己给自己授权**，必须你手动在「系统设置」里开。
> 伯乐的「设置 → 一键修复」里有直达按钮，点一下就能跳到下面每个面板。

在 **系统设置 → 隐私与安全性** 里，给「**运行伯乐的程序**」（Terminal 或伯乐.app / Chrome）打开：

1. **辅助功能（Accessibility）** —— 否则识别不了页面元素
2. **屏幕录制（Screen Recording）** —— 否则读不到页面内容
3. **自动化（Apple 事件）** —— 允许控制 Chrome

再在 **Chrome 菜单栏 → 显示 → 开发者 → ☑️ 允许来自 Apple 事件的 JavaScript**。

## 第 5 步 · 启动伯乐

拿到项目文件夹后，任选一种：

| 方式 | 操作 |
|------|------|
| **原生 App**（推荐） | 终端在项目目录运行 `bash desktop/build_app.sh` 生成 `伯乐.app` → 之后双击它 |
| **双击脚本** | 双击 `desktop/伯乐.command` |
| **命令行** | `python3 desktop/server.py` |

浏览器会自动弹出伯乐界面（`http://127.0.0.1:8765`）。

## 第 6 步 · 登录 + 自检 + 配 Key

1. **登录**：勾选同意《隐私与数据安全条款》，用发你的账号登录。
2. **设置 → 装机自检**：看是不是全绿；有缺的点「**一键修复**」（缺配置会自动补，权限项给你直达链接）。
3. **设置 → DeepSeek API**：把 Key 填进去，点「**测试连接**」，看到「连接正常」即成功（不填会降级成模板话术）。

✅ 自检关键项全绿 + DeepSeek 测试通过 = 装好了。接着看《[使用教程](使用教程.md)》。

## 常见问题

- **双击 `.app` 打不开 / 报错 -10669**：先用 `bash desktop/build_app.sh` 重新生成；仍不行就用 `伯乐.command` 或命令行启动。首次被拦：右键→打开一次。
- **自检显示「未登录」**：还没登录账号，或登录态过期，重新登录。
- **一直是模板话术、不智能**：DeepSeek Key 没配或失效 → 设置页填入 + 测试连接。
- **操作没反应 / 识别失败**：多半是第 4 步的权限没开全，或 BOSS 没登录。回设置页看自检的人工确认项。
- **端口被占**：伯乐会自动换端口（8765→8776），或用 `--port` 指定。
