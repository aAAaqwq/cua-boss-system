# 权限架构（谁能做什么 · 用哪把钥匙）

> 配套 [数据安全说明.md](数据安全说明.md)。一句话：**普通用户登录后只能动自己的数据；管理员能管全部；万能钥匙只在服务端。**

## 1. 三类角色
| 角色 | 是谁 | 能做什么 | 凭什么 |
|------|------|---------|--------|
| **普通用户** | 你开通账号下发给他 | 在自己机器跑 agent 采集；网页**只看自己**的数据 | 账号密码 → 登录拿 **user token**（RLS 限自己 tenant）|
| **管理员** | `admin@...`（uid 在白名单） | 网页后台**管所有用户+数据**、开通/停用账号 | 同样账号登录 → Worker 校验是管理员 → 用 service_role 代办 |
| **拥有者(你)** | 持有全部密钥 | 一切；本地 `admin.py` 管理；建表/取key | `.env` 里的 service_role + PAT |

## 2. 四把钥匙（关键）
| 钥匙 | 作用 | 谁持有 / 在哪 | 敏感度 |
|------|------|-------------|--------|
| **service_role** | **万能钥匙**：绕过 RLS，读写**所有人**的数据、建/封账号 | 拥有者 `.env` + 管理 Worker(服务端 secret) | 🔴 绝密，**绝不进浏览器/用户机器/git** |
| **anon key** | 「入口钥匙」：登录、前端调用都用它 | `.env` + 前端(公开) | 🟢 公开安全（本身看不到任何数据，受 RLS 管）|
| **user token** | 登录后换取，代表「我是某用户」 | 用户/管理员本机(`data/.cloud_auth.json`)、浏览器内存 | 🟡 个人凭证，会过期 |
| **PAT** | 账号级管理（建表、取 key） | 拥有者 `.env` | 🔴 账号级，**用完应 Revoke** |

## 3. 架构图（数据流 + 鉴权）
```
普通用户机器                         Supabase (Postgres + Auth + RLS)
  agent ──登录(anon)──► user token ──┐
  agent ──带 user token 上传────────►│ candidates 表
                                     │  RLS: tenant_id = auth.uid()
用户浏览器 ──登录──► 看自己数据 ◄──────┘  (只放行自己那行)
                                     ▲
管理员浏览器 ──登录──► user token ──► Cloudflare Worker(持 service_role)
  (admin.html)                        校验 token + uid∈白名单 → 用 service_role
                                        → 管所有用户/数据(绕过 RLS)
拥有者本机 ──service_role/PAT──► admin.py(CLI) 直管
```

## 4. 三道安全门
1. **登录门禁**：agent 启动即 `require_account()`——**没有我们下发的账号、没登录 → 拒绝运行**（用户机器无 service_role，绕不过）。
2. **RLS 行级隔离**：数据库强制 `tenant_id = auth.uid()`——登录用户**只能读写自己 tenant 的行**，跨用户零可见（实证：anon 未登录读 = 0 行）。
3. **管理员白名单**：管理后台 Worker 校验登录者 uid 必须 ∈ `ADMIN_UIDS`，否则 403——**普通用户登进管理页也管不了别人**。service_role 只在 Worker 服务端，浏览器永不接触。

## 5. service_role 到底是什么（你问的那把 key）
- 它是 Supabase 的**超级管理员密钥**，**无视 RLS**——拿着它能读写**所有用户**的全部数据、建号封号。
- **为什么管理后台需要它**：管理员要「看所有人、管所有人」，必须绕过 RLS，只有 service_role 能做。
- **为什么必须藏在 Worker 服务端**：一旦进了浏览器/用户机器/git，等于把整库交出去。所以它只作为 **Cloudflare Worker 的 secret** 存在云端服务器，管理员浏览器只拿临时 user token，由 Worker 在服务端代持 service_role 执行。
- **为什么 AI（我）不能帮你设它**：把这把万能钥匙发往第三方(Cloudflare)属于凭证外泄，平台硬性禁止 AI 代做——只能你本人执行。

## 6. 拥有者管理日常用什么
- **网页后台** `bole-hr.pages.dev/admin.html`（管理员登录）—— 推荐，可视化管用户+数据。
- **CLI** `scripts/admin.py`（本机 service_role）—— `users / create / disable / enable / data`。
- **PAT** 仅建表/取 key 用，平时用不到，**验证完应 Revoke**。
