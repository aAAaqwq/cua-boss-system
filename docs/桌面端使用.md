# 伯乐桌面端使用说明

给不懂技术的 HR 的图形界面。双击即用，不碰终端、不看代码。

> 面向用户的产品人设见 [../IDENTITY.md](../IDENTITY.md)；技术细节见 [../CLAUDE.md](../CLAUDE.md)。

## 一、这是什么

一个 macOS 桌面 App「**伯乐**」——本地 Web App 形态：一个只监听 `127.0.0.1` 的纯标准库
本地服务，把浏览器界面桥接到既有招聘脚本与候选人数据库。**这一层就是将来 Tauri 壳的
WebView 前端**（打包时直接复用，不重写）。

```
浏览器界面(desktop/ui) ── HTTP/127.0.0.1 ── 本地服务(desktop/server.py)
                                                   │
                          ┌────────────────────────┼────────────────────────┐
                       招聘脚本(scripts/*)      候选人库(data/candidates.db)   DeepSeek/Supabase
                       cua-driver 驱动 Chrome
```

## 二、三种启动方式

| 方式 | 命令/操作 | 特点 |
|------|-----------|------|
| 原生 App | `bash desktop/build_app.sh` 生成 `伯乐.app` → 双击 | 独立图标、无终端窗口，最像成品 |
| 双击脚本 | 双击 `desktop/伯乐.command` | 起服务 + Chrome「应用窗口」模式；关掉终端不影响 |
| 命令行 | `python desktop/server.py`（`--port` / `--no-open` 可选） | 开发/调试 |

启动后浏览器打开 `http://127.0.0.1:8765/`。**退出**：界面左下「关闭程序」一键停服（会先停掉正在跑的任务），无需终端。

## 三、首次上手

1. **登录**：用下发的账号登录（登录态存本机 `data/.cloud_auth.json`）。登录前需勾选同意《隐私与数据安全条款》。
2. **装机自检**（设置页）：绿灯才好开跑。会真实体检 Python / cua-driver / Chrome / DeepSeek Key / 配置 / 登录 / 数据库，并列出「必须人工确认」项（BOSS 已登录、辅助功能+屏幕录制授权、Apple 事件 JS、防睡眠）。
3. **配 DeepSeek Key**（设置页）：填入即写 `.env` 且即时生效（不用重启），点「测试连接」验证。不配则智能回复/评分降级为模板原文。

## 四、四个页面

- **看板**：人才漏斗统计 + 评分榜；**点候选人卡片**看完整档案（个人信息 / 简历 PDF 预览 / AI 评分维度与依据 / 聊天记录）。
- **操作台**：打招呼 / 收简历 / 智能沟通 / 全流程，一键后台跑，**实时日志**逐行输出；可 dry-run 预览、设最低学历。
- **问伯乐**：像聊天一样说需求，伯乐给出下一步；快捷回复按伯乐的回答**动态生成**，点即发。
- **设置**：DeepSeek Key / 模型 / 接口 / 云同步开关 + 测试连接 + 装机自检。

## 五、执行端代码在不在「安装包」里？

**看你怎么装：**

- **当前形态（源码分发 / GitHub Release 源码包）**：**是，全部执行端代码都在里面。**
  发布的源码归档（`Source code (zip/tar.gz)`）包含完整引擎——`scripts/`（打招呼/收集/沟通/评分等自动化）、
  `app/`（DeepSeek/评分/数据库/云同步）、`desktop/`（界面+本地服务）、`config/`（话术/筛选/评分配置模板）。
  解压后即完整可跑。**前提**：本机需已装 Python 3.10+ 与 cua-driver。
- **原生 `伯乐.app`（osacompile 生成）**：**不含**执行端代码。它只是一个**启动器**——把仓库的绝对路径烧进去，
  双击时去调用仓库里的 `desktop/server.py`。所以 `.app` 必须和仓库放在一起、且本机有 Python，才能跑；
  它**不是**自包含安装包（移机需重跑 `build_app.sh`）。
- **真正的自包含安装包（P3，未完成）**：把 **Python 运行时 + 全部执行端代码**一起打进 `.app`，再 Apple
  签名公证成 `.dmg`。用户零依赖、拖进「应用程序」即用。需要 Apple 开发者证书（$99/年）+ py2app/PyInstaller/Tauri
  打包链。见 [产品化规划.md](产品化规划.md) 第六节。

**一句话**：`v1.0` 的 GitHub Release 源码包**已含全部执行端代码**；缺的只是「把 Python 运行时也塞进去、免安装」这一步（P3）。

## 六、常见问题

- **原生 App 双击打不开 / 报 -10669**：新版 macOS 拒绝未签名的手搓脚本包，故改用 `osacompile`；仍打不开就用 `伯乐.command` 或命令行。首次可能被 Gatekeeper 拦，右键→打开一次。
- **端口被占**：服务会自动往后找空闲端口（8765→8776）；或 `--port` 指定。
- **智能回复/评分是模板原文**：DeepSeek Key 没配或失效，去设置页填入并「测试连接」。
- **操作没反应/AX 识别失败**：多为人工确认项没就绪——BOSS 未登录、辅助功能/屏幕录制未授权、Apple 事件 JS 未开。回设置页看自检。
- **数据在哪**：候选人库 `data/candidates.db`、简历 PDF `data/resumes/`、登录态 `data/.cloud_auth.json`（均本机、gitignore）。

## 七、安全边界

服务仅绑回环地址；跨站请求校验 Origin（防 CSRF）；未登录不返回任何候选人 PII；简历路径限定 `data/resumes/` 内；后台任务白名单；`.env` 写入拒控制字符。详见 [数据安全说明.md](数据安全说明.md)。
